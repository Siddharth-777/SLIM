API highlights

Prediction Dashboard Anomaly Research Simulation Alerts
Forecast — `POST /api/forecast` returns next-interval water quality predictions and structured metadata per target column.
Dashboards — `GET /api/lake-data/history` hydrates live charts with denoised, interpolated sensor streams and rolling stats.
Anomaly insights — `POST /api/analyze` adds root-cause hypotheses and remediation tips alongside anomaly scores.
Digital twin — `POST /api/digital-twin` simulates warming spikes, pollution slugs, and rainfall recovery scenarios for planning.
Alerts & events — `POST /api/event-detection` classifies polluted inflow, rain-driven turbidity, and aerator risks with urgency levels.

Prediction

Temporal Fusion Transformers forecast the next readings for pH, turbidity, temperature, and dissolved oxygen so teams can act ahead of critical thresholds.

Signal-aware checkpoints

Dedicated TFT checkpoints per sensor stream keep forecasts tailored to each characteristic while sharing common dataset normalization.

CSV + live blend

History is hydrated from Supabase while backfilling from local CSV archives, ensuring continuity even if connectivity blips occur.

Readable outputs

Responses return ISO8601 timestamps plus a full LakeReading payload, making it simple to populate dashboards or trigger workflows.

Prediction dashboard

Live tiles, rolling averages, and correlation callouts give stakeholders a real-time pulse on lake health.

Live ingestion

Continuous `POST /api/lake-data` writes to Supabase with API-key validation and resilient caching of the client session.

Flexible queries

`GET /api/lake-data/latest` and `GET /api/lake-data/history` power tiles, trends, and drill-downs without extra ETL.

Storytelling analytics

`POST /api/data-query` pairs Groq LLM reasoning with dataset statistics to narrate patterns in natural language.

Anomaly detection

Tri-layer anomaly checks blend edge screening, ML heuristics, and advisory context to cut through noise.

Structured response

`POST /api/analyze` returns severity, narrative context, and recommended mitigations alongside the raw anomaly score.

Context-rich triage

Feature-driven heuristics highlight likely drivers (sewage inflow, aerator failure, rainfall) to focus operator attention.

Interoperable by design

JSON payloads mirror the LakeReading schema, simplifying how dashboards overlay alerts on live streams.

Anomaly research

Exploratory modeling endpoints accelerate hypothesis testing and model comparison.

Clustering patterns

`POST /api/clusters` surfaces centroids and segments, revealing recurring profiles such as clear mornings vs. post-rain spikes.

Relationship insights

`POST /api/relationships` quantifies how features co-move, enabling policy simulations for aeration, inflow diversion, or dosing.

Research models

`POST /api/research-models` compares classic baselines to generative signals, returning metrics ready for executive readouts.

Digital twin simulation

Simulate pollution slugs, warming spikes, and rainfall-driven turbidity recovery to test interventions before rolling them out.

Scenario library

`POST /api/digital-twin` accepts pollution intensity, rain strength, and recovery rate to project different what-if pathways.

Transparent curves

Outputs include time-indexed trajectories for turbidity and DO, perfect for overlaying on operational dashboards.

Actionable summaries

Each scenario ships with short narratives explaining operational risk, remediation options, and projected stabilization times.

Alerts & event detection

Human-readable alerts ensure teams know what happened, why it matters, and how to respond.

Label-free checks

`POST /api/event-detection` identifies polluted inflows, rain turbidity, and aerator risks using contextual thresholds instead of labels.

Severity tagging

Each alert carries an urgency label and supporting evidence so incident commanders can triage faster.

Secure logging

Supabase-backed storage plus optional blockchain integrity layers keep event timelines auditable end-to-end.
